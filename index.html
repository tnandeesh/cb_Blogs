Amazon EC2 Auto Scaling

Amazon EC2 Auto Scaling helps you ensure that you have the correct number of Amazon EC2 instances available to handle the load for your application. You create collections of EC2 instances, called Auto Scaling groups. You can specify the minimum number of instances in each Auto Scaling group, and Amazon EC2 Auto Scaling ensures that your group never goes below this size. You can specify the maximum number of instances in each Auto Scaling group, and Amazon EC2 Auto Scaling ensures that your group never goes above this size. If you specify the desired capacity, either when you create the group or at any time thereafter, Amazon EC2 Auto Scaling ensures that your group has this many instances. If you specify scaling policies, then Amazon EC2 Auto Scaling can launch or terminate instances as demand on your application increases or decreases.

For example, the following Auto Scaling group has a minimum size of one instance, a desired capacity of two instances, and a maximum size of four instances. The scaling policies that you define adjust the number of instances, within your minimum and maximum number of instances, based on the criteria that you specify.

Creating an AWS Auto Scaling Group

1. Navigate to Auto Scaling:

  • In the AWS Management Console, go to the "Auto Scaling" service.

    • 2. Create a Launch Configuration:

  • Click on "Create launch configuration."
  • Choose the Amazon Machine Image (AMI) for your instances.
  • Select an instance type, add user data if needed, and configure storage options.
  • Configure security groups and any other settings required for your instances.

    • 3. Create an Auto Scaling Group:

  • After creating the launch configuration, go back to the "Auto Scaling" dashboard and click on "Create Auto Scaling group."
  • Define your group name, group size, and network settings.
  • Configure scaling policies, specifying when to scale out (add instances) or scale in (remove instances).
  • Set up health checks to determine the instance's health.

    • 4. Configure Scaling Policies:

  • Define policies based on CloudWatch alarms or other metrics.
  • For example, you can create a policy to scale out when CPU utilization exceeds a certain threshold.

    • 5. Configure Notifications (Optional):

  • Set up Amazon Simple Notification Service (SNS) notifications to receive alerts when scaling events occur.

    • 6. Review and Launch:

  • Review your configurations and click "Create Auto Scaling group."

    • 7. Monitor and Test:

  • Monitor your Auto Scaling group in the AWS Management Console.
  • Test the Auto Scaling group by manually terminating instances to see if new instances are launched to replace them.

    • 8. Adjust as Needed:

  • Adjust your Auto Scaling group settings as needed, such as updating the launch configuration, modifying scaling policies, or adjusting health check parameters.

    Pricing for Amazon EC2 Auto Scaling

    There are no additional fees with Amazon EC2 Auto Scaling, so it's easy to try it out and see how it can benefit your AWS architecture.

    You only pay for the AWS resources (for example, EC2 instances, EBS volumes, and CloudWatch alarms) that you use.
